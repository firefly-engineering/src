# Phase 0.5: Implement Buck2 Config Generation

## Overview

Implement Buck2 configuration file generation that references the same resolved toolchain derivations used in the development shell. This completes the synchronization loop.

## Context

Buck2 needs to know where to find system toolchains. We generate Buck2 configuration files that point to the **exact same Nix store paths** used in the development shell.

### Synchronization Architecture

```
Resolved Toolchains
{ go = /nix/store/abc-go-1.21.5;
  rust = /nix/store/xyz-rust-1.75.0; }
        │
        ├─→ Shell: packages = [/nix/store/abc-go-1.21.5, ...]
        │   $ which go → /nix/store/abc-go-1.21.5/bin/go
        │
        └─→ Buck2: [go_bin] → /nix/store/abc-go-1.21.5/bin/go
            $ buck2 audit config go_bin → /nix/store/abc-go-1.21.5/bin/go

✅ Same path = Guaranteed synchronization
```

When toolchain version changes:
- Nix store path changes (content-addressed)
- Buck2 config regenerates with new path
- Buck2 cache automatically invalidates (different path = different cache key)

## Implementation Guidance

### 1. Buck2 Configuration Structure

Buck2 can be configured via:
- `.buckconfig` - Main configuration file
- `toolchains/BUCK` - Toolchain platform definitions
- Per-language config files

We'll generate toolchain-specific config:

```
repo/
├── .buckconfig                    # Existing, points to generated configs
├── .buckconfig.toolchains         # Generated - toolchain paths
└── toolchains/                    # Generated - Buck2 toolchain definitions
    ├── BUCK
    ├── go.bzl
    ├── rust.bzl
    └── python.bzl
```

### 2. Generate .buckconfig.toolchains

Create a function to generate Buck2 config:

```nix
let
  # Generate toolchain configuration
  generateBuckConfig = resolved:
    let
      entries = lib.mapAttrsToList (name: derivation:
        if name == "go" then ''
          [go]
          go_bin = ${derivation}/bin/go
        ''
        else if name == "rust" then ''
          [rust]
          rustc_bin = ${derivation}/bin/rustc
          cargo_bin = ${derivation}/bin/cargo
        ''
        else if name == "python" then ''
          [python]
          python_bin = ${derivation}/bin/python
        ''
        else if name == "clang" then ''
          [cxx]
          cc = ${derivation}/bin/clang
          cxx = ${derivation}/bin/clang++
        ''
        else ""
      ) resolved;
    in
    lib.concatStringsSep "\n" entries;

  buckConfigContent = ''
    # Generated by firefly-toolchains module
    # DO NOT EDIT - This file is automatically generated
    # Source: toolchain.toml via Nix flake

    ${generateBuckConfig resolved}
  '';
in
```

### 3. Generate toolchains/BUCK

Create Buck2 toolchain platform definitions:

```nix
let
  generateToolchainsBuck = resolved: ''
    # Generated by firefly-toolchains module
    # DO NOT EDIT

    load("@prelude//toolchains:genrule.bzl", "system_genrule_toolchain")
    load(":go.bzl", "system_go_toolchain")
    load(":rust.bzl", "system_rust_toolchain")

    ${lib.optionalString (resolved ? go) ''
    system_go_toolchain(
        name = "go",
        visibility = ["PUBLIC"],
    )
    ''}

    ${lib.optionalString (resolved ? rust) ''
    system_rust_toolchain(
        name = "rust",
        visibility = ["PUBLIC"],
    )
    ''}

    genrule_toolchain(
        name = "genrule",
        visibility = ["PUBLIC"],
    )
  '';
in
```

### 4. Generate Language-Specific Files

Go toolchain definition (`toolchains/go.bzl`):

```nix
let
  generateGoToolchain = goDerivation: ''
    def system_go_toolchain(**kwargs):
        return go_toolchain(
            go = "${goDerivation}/bin/go",
            **kwargs
        )
  '';
in
```

Rust toolchain definition (`toolchains/rust.bzl`):

```nix
let
  generateRustToolchain = rustDerivation: ''
    def system_rust_toolchain(**kwargs):
        return rust_toolchain(
            rustc = "${rustDerivation}/bin/rustc",
            cargo = "${rustDerivation}/bin/cargo",
            **kwargs
        )
  '';
in
```

### 5. Write Generated Files

Use Nix to write files during shell activation:

```nix
let
  # Activation script to generate Buck2 configs
  generateConfigs = pkgs.writeScriptBin "generate-buck2-configs" ''
    #!/usr/bin/env bash
    set -euo pipefail

    echo "Generating Buck2 toolchain configurations..."

    # Create toolchains directory
    mkdir -p toolchains

    # Generate .buckconfig.toolchains
    cat > .buckconfig.toolchains <<'EOF'
    ${buckConfigContent}
    EOF

    # Generate toolchains/BUCK
    cat > toolchains/BUCK <<'EOF'
    ${generateToolchainsBuck resolved}
    EOF

    ${lib.optionalString (resolved ? go) ''
      cat > toolchains/go.bzl <<'EOF'
      ${generateGoToolchain resolved.go}
      EOF
    ''}

    ${lib.optionalString (resolved ? rust) ''
      cat > toolchains/rust.bzl <<'EOF'
      ${generateRustToolchain resolved.rust}
      EOF
    ''}

    echo "✅ Buck2 configs generated"
  '';
in
{
  devShells.default = pkgs.mkShell {
    packages = toolchainPackages ++ [ generateConfigs ];

    shellHook = ''
      # Auto-generate configs on shell entry if enabled
      ${lib.optionalString cfg.buck2.autoGenerate ''
        generate-buck2-configs
      ''}
    '';
  };
}
```

### 6. Update .buckconfig

Add include directive in main `.buckconfig`:

```ini
[buildfile]
name = BUCK

[repositories]
prelude = prelude
root = .

# Include generated toolchain configuration
<file:.buckconfig.toolchains>
```

### 7. Module Options

Add configuration options:

```nix
{
  options.firefly.toolchains.buck2 = {
    enable = lib.mkOption {
      type = lib.types.bool;
      default = true;
      description = "Enable Buck2 config generation";
    };

    autoGenerate = lib.mkOption {
      type = lib.types.bool;
      default = true;
      description = "Automatically generate configs on shell entry";
    };

    outputDir = lib.mkOption {
      type = lib.types.path;
      default = ./.;
      description = "Directory to write generated configs";
    };
  };
}
```

### 8. Gitignore Generated Files

Create/update `.gitignore`:

```gitignore
# Generated toolchain configurations
.buckconfig.toolchains
toolchains/BUCK
toolchains/*.bzl

# But keep custom toolchain definitions if any
!toolchains/custom/
```

## Testing

```bash
# Enter shell (should auto-generate configs)
nix develop

# Verify files were generated
ls -la .buckconfig.toolchains
ls -la toolchains/

# Test Buck2 can read configs
buck2 audit config go

# Verify path matches shell
GO_SHELL=$(which go)
GO_BUCK2=$(buck2 audit config go_bin)

echo "Shell:  $GO_SHELL"
echo "Buck2:  $GO_BUCK2"

# Should be identical!
[ "$GO_SHELL" = "$GO_BUCK2" ] && echo "✅ Synchronized!" || echo "❌ Mismatch!"

# Test regeneration
rm .buckconfig.toolchains
generate-buck2-configs
# Should recreate file
```
